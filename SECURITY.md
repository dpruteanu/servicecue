# Security Policy

ServiceCue is a local-only desktop app. It should not require internet access during service and should not upload church media files.

## Supported Versions

Security updates are provided for the latest published release.

## Reporting A Vulnerability

Please report security issues privately to the project maintainer instead of opening a public issue.

If the repository owner has GitHub private vulnerability reporting enabled, use GitHub's **Report a vulnerability** button. Otherwise, contact the maintainer directly.

Please include:

- A clear description of the issue.
- Steps to reproduce it.
- The operating system and ServiceCue version.
- Whether church media files or local paths could be exposed.

## Sensitive Data

Do not include private church media, schedules, personal names, or local file paths in public bug reports unless they are clearly anonymized.
